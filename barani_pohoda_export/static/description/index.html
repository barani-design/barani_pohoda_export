BARANI POHODA Export

Export customer invoices, tax-configuration-driven advance (down-payment) invoices, settlement invoices and approved credit notes from Odoo 16 to POHODA XML (Slovak / Czech accounting by STORMWARE).

Knows your documents

A document classifier reads native Odoo accounting facts and tells regular invoices, advance invoices (324), their settlements and credit notes apart — including the advance-deduction lines a final invoice carries.

Mapping matrix

Export rules × fiscal profiles, edited in a visual matrix: POHODA předkontace, VAT classification, control-statement and MOSS codes per cell, with states (active / blocked / review) and inline warnings. CSV export for the accountant.

Blocks instead of guessing

A read-only preflight aggregates explicit blockers — unmapped lines, missing codes, advance/VAT misconfigurations, wrong company, unposted documents — so nothing questionable ever reaches POHODA.

Audited lifecycle

Each export batch archives the request and response XML with SHA-256 hashes, mapping/configuration version hashes and per-document results. Batches beyond draft are immutable audit records.

XSD validation

Optionally validate every generated dataPack against the official STORMWARE schema set (single .xsd or the full .zip) before a human ever touches it.

Closes the loop

Import POHODA's response file and every document gets its accepted / warning / error verdict, the produced record number, and a status mirror right on the invoice — with a managed, reason-tracked re-export path.

How an export runs

The mapping matrix

One advance document kind; VAT comes from your Odoo tax configuration (0% for EU B2B / export, positive for domestic §19(4) advances). The matrix decides the POHODA codes per rule, per fiscal profile, per document kind.

Designed to be verified, not trusted blindly

All mapping values ship as migration seeds that your accountant must review (the seed rules are intentionally inactive). Run a POHODA import test on a trial company before production. Phase 1 performs no automatic sending: you download the XML, import it into POHODA yourself, and feed the response back. Credit notes stay gated until your import test clears them.

Getting started

  1. Install the app and open POHODA Export → Configuration.
  2. Review the code dictionaries (předkontace, VAT, KV, MOSS) with your accountant.
  3. Link your fiscal positions to fiscal profiles and activate the export rules.
  4. Fill the Mapping Matrix; resolve every review-required cell.
  5. Run New Export for a test period, import the file into POHODA, and bring the response back.

Requirements

  • Odoo 16 (Community or Enterprise), account + sale_management.
  • Python lxml (ships with Odoo).
  • STORMWARE POHODA with XML import enabled (SK / CZ).
  • Roles: Export User (read), Export Manager (run exports), Configuration Manager (matrix & dictionaries).
